# Guru Approve Task Plan

Use this Skill after current
`guru-review-contract-wording:planning_artifacts:pass` evidence exists and
before task activation. Load [references/contract.md](references/contract.md)
before acting.

Validate all nine entry preconditions. Review the three planning documents and
their `Docs SSOT Plan`, cover every load-bearing item with one provenance class,
review implementation choices and unusual-scenario proposals, revise
task-internal gaps, route authority changes to `guru-clarify-requirements`, and
complete the final AI Review Gate. Obtain dedicated proposal confirmation only
when triggered. An ordinary approved scope expansion binds a controlled locator
in one planning artifact; an unusual expansion binds one canonical unusual
candidate. In both cases the dedicated confirmation and current authority must
bind the same runtime-recomputed proposal digest. Then separately display all
three planning links and obtain explicit post-planning confirmation before
recording an `approved` result.

Call the recorder and checker only after the semantic result exists. They
validate objective facts, schema, digests, freshness, and the closed
Gate/exit/consumer union; they never decide adequacy, provenance, proposal
necessity, confirmation sufficiency, semantic pass, or route. Return exactly one
of `approved`, `revision_required`, `clarify_scope`, or `blocked`. Fail
closed when evidence or the compatible full Guru Team preset runtime is
missing. This package is not self-contained or portable.
